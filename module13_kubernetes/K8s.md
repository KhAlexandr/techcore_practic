**Pod** - Наименьшая и простейшая единица в Kubernetes. Представляет собой группу из одного или нескольких контейнеров с общими сетевыми пространствами и хранилищем.

**Deployment** - Ресурс высшего уровня, который управляет жизненным циклом приложений, обеспечивая декларативные обновления, масштабирование и отказоустойчивость подов.

**Service** - Абстракция, которая определяет логический набор подов и политику доступа к ним. Обеспечает стабильную конечную точку для сетевого доступа.

**Node** - Рабочая машина в Kubernetes, которая может быть виртуальной или физической. На узлах запускаются поды.

**Cluster** - Набор узлов, объединенных для обеспечения высокой доступности и распределения нагрузки.

**ConfigMap** - Объект для хранения нефункциональных данных в формате ключ-значение, отделяющий конфигурацию от образов контейнеров.

**Secret** - Аналогичен ConfigMap, но предназначен для хранения конфиденциальной информации в закодированном виде.

**Ingress** - API-объект, который управляет внешним доступом к сервисам в кластере, обычно через HTTP/HTTPS.

**kubectl** - Командный интерфейс (CLI) для управления кластерами Kubernetes. Позволяет развертывать приложения, инспектировать и управлять ресурсами кластера через API Kubernetes.

Основные функции:
- Создание и удаление ресурсов (поды, развертывания, сервисы)
- Просмотр состояния кластера и приложений
- Масштабирование и обновление развертываний
- Отладка и диагностика проблем
